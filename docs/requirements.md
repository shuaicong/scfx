## 1. 项目概述

### 1.1 项目背景
海南储备集团粮食市场智能分析平台旨在自动化采集国内外粮食市场数据，建立企业知识库，提供 AI 智能问答和自动生成分析报告。

### 1.2 项目目标
- 实现 8+ 数据源的自动化采集（粮信网、我的钢铁网、中华粮网等）
- 建立企业级粮食行业知识库
- 提供 AI 智能问答（价格、走势、查询分析）
- 自动生成周报、月报（带图表）

## 3. 采集内容类型

### 3.1 报告类
- **晨报**: 每日市场早报
- **日报**: 每日市场日报
- **周报**: 每周市场总结
- **月报**: 每月市场分析
- **专题报告**: 专项研究分析

### 3.2 资讯类
- **市场动态**: 价格波动、供需变化
- **政策解读**: 粮食政策影响分析
- **行业新闻**: 产业链相关信息

### 3.3 价格数据
- **现货价格**: 各地区现货成交价
- **期货价格**: 期货合约收盘价
- **历史价格**: 历史价格走势数据

## 4. 功能需求

### 4.1 采集任务管理
- 创建、编辑、删除采集任务
- 任务定时调度（Cron 表达式）
- 手动触发立即执行
- 任务优先级设置
- 任务执行超时控制
- 失败自动重试（可配置次数）

### 4.2 数据采集（Python）
- 模拟登录（Cookie/Session 持久化）
- 页面自动化导航
- 数据解析（HTML/JSON/XML）
- 文件下载（PDF、图片）
- 增量采集（判断是否重复）
- 异常处理与日志记录

### 4.3 数据存储
- 原始数据文件存储
- 结构化数据入库
- 向量化存储（知识库）
- 数据去重检查

### 4.4 知识库
- 向量数据库存储
- 相似度检索
- RAG（检索增强生成）

### 4.5 AI 智能问答
- 自然语言查询
- 价格走势分析
- 供需情况问答
- 政策影响解读

### 4.6 报告生成
- 周报自动生成
- 月报自动生成
- 自定义报告模板
- 图表生成（ECharts）

### 4.7 监控告警
- 任务执行状态监控
- 成功率统计
- 失败告警（邮件/钉钉/微信）
- 数据异常告警

### 4.8 仪表板
- 各数据源统计（分门别类）
- 今日/本周/本月数据概览
- 采集日志实时查看
- 告警信息汇总
- 快速操作入口

## 5. 性能需求

### 5.1 采集性能
- 单页面解析 < 3 秒
- 单任务超时时间可配置（默认 300 秒）
- 支持任务并发数控制（避免被封禁）

### 5.2 系统性能
- API 响应时间 < 200ms
- 仪表板加载时间 < 2 秒
- 支持 100+ 并发用户

## 6. 非功能性需求

### 6.1 安全性
- JWT 认证
- 敏感信息加密存储
- 采集频率限制

### 6.2 可用性
- 7x24 小时运行
- H2 开发模式 / MySQL 生产模式
- Docker 容器化部署

### 6.3 可维护性
- 模块化设计
- 统一日志记录
- 配置外部化

## 7. 技术栈

### 7.1 后端
- Java 17
- Spring Boot 3.2
- MyBatis-Plus 3.5
- H2 Database（开发）/ MySQL（生产）

### 7.2 前端
- Vue 3 + TypeScript
- Element Plus
- ECharts
- Vite

### 7.3 采集器
- Python 3.10+
- Playwright
- requests

### 7.4 中间件
- Redis（可选，缓存）
- 向量数据库（Milvus/Pinecone，可选）
