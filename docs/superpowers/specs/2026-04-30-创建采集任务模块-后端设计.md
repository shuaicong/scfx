# 创建采集任务模块 - 后端系统设计

## 1. 模块概述

### 1.1 前端原型功能
根据 `create-task-prototype.html` 前端原型，本模块需要支持：

- **任务基本信息**：名称、描述
- **脚本配置**：
  - 单文件脚本（编辑器/上传）
  - 多文件包（zip上传 + 入口文件）
- **触发配置**：
  - 单次触发（指定日期时间）
  - 周期触发（每天/每周/每月）
  - Cron表达式（支持5-8位格式）
- **操作**：保存、保存并执行

### 1.2 后端职责
- 任务 CRUD 操作
- 脚本文件存储（本地磁盘）
- 触发配置解析和调度
- 脚本版本管理
- 执行记录管理

  "time": "08:00:00"

  "repeatType": "monthly",

  "time": "08:00:00"
}

// Cron表达式配置
{
  "type": "cron",
  "expression": "0 0 8 * * *"
}

## 3. API 设计

### 3.1 接口列表

| 方法 | 路径 | 说明 | 请求体 |
|------|------|------|--------|
| POST | `/tasks` | 创建任务 | 任务信息 |
| PUT | `/tasks/{id}` | 更新任务 | 任务信息 |
| GET | `/tasks/{id}` | 获取任务详情 | - |
| DELETE | `/tasks/{id}` | 删除任务 | - |
| POST | `/tasks/{id}/execute` | 执行任务 | - |
| POST | `/tasks/validate-cron` | 校验Cron表达式 | cron表达式 |

### 3.2 创建任务 API

```
POST /tasks

Request:
{
  "taskName": "粮信网玉米晨报采集",
  "taskDesc": "定时采集粮信网玉米价格数据",
  "scriptType": "single",           // single | multi
  "scriptContent": "...",           // 单文件脚本内容
  "scriptFile": "...",              // 或上传的文件名
  "zipFile": "...",                 // 多文件包文件名
  "entryFile": "main.py",           // 入口文件（多文件包时必填）
  "triggerType": "cycle",           // once | cycle | cron
  "triggerConfig": {               // 见上文数据结构
    "type": "weekly",
    "days": [1, 2, 3, 4, 5],
    "time": "08:00:00"
  }
}

Response:
{
  "code": 200,
  "data": {
    "id": 1,
    "taskName": "粮信网玉米晨报采集",
    "status": "pending",
    "currentVersion": 1,
    "createdAt": "2026-04-30T12:00:00"
  }
}
```

## 4. Service 层设计

### 4.1 CollectionTaskService

```java
public interface CollectionTaskService {
    // 创建任务
    Result<CollectionTask> createTask(CreateTaskRequest request);

    // 更新任务（自动创建新版本）
    Result<CollectionTask> updateTask(Long id, UpdateTaskRequest request);

    // 获取任务详情
    Result<CollectionTask> getTaskById(Long id);

    // 删除任务
    Result<Void> deleteTask(Long id);

    // 执行任务
    Result<Map<String, Object>> executeTask(Long id);

    // 校验Cron表达式
    Result<CronValidationResult> validateCron(String cron);
}
```

### 4.2 核心流程

#### 创建任务流程
```
1. 校验请求参数（任务名、脚本、触发配置）
2. 保存脚本文件到本地磁盘
   - 单文件: scripts/{taskId}/v1/main.py
   - 多文件: scripts/{taskId}/v1/{filename}.zip
3. 创建任务记录
4. 创建初始版本记录
5. 如果触发类型为周期/Cron，注册调度任务
6. 返回任务ID
```

#### 更新任务流程
```
1. 校验任务是否存在
2. 如果脚本内容变更:
   - 保存新脚本文件到新版本目录
   - scripts/{taskId}/v{version+1}/main.py
   - 创建新版本记录
3. 更新任务元数据
4. 重新注册调度任务
5. 返回更新后的任务
```

#### 执行任务流程
```
1. 获取任务当前版本的脚本内容
2. 创建执行记录 (t_task_execution)
3. 调用 Python 执行器执行脚本
4. 异步返回执行结果
```

## 5. 文件存储设计

### 5.1 目录结构

```
scripts/
├── {taskId}/
│   ├── v1/
│   │   ├── main.py              # 单文件脚本
│   │   └── package.zip          # 多文件包压缩包
│   ├── v2/
│   │   └── main.py
│   └── ...
```

### 5.2 存储策略

| 场景 | 存储方式 |
|------|----------|
| 单文件脚本 | 保存为 `main.py` |
| 多文件包 | 保存原始 `.zip` 文件，解压后执行入口文件 |
| 版本目录 | 按版本号隔离 `{taskId}/v{version}/` |

## 6. Cron 表达式解析

### 6.1 支持格式

| 字段数 | 格式 | 说明 |
|--------|------|------|
| 5位 | 分 时 日 月 周 | 标准Unix cron（无秒） |
| 6位 | 秒 分 时 日 月 周 | Quartz格式 |
| 7位 | 秒 分 时 日 月 周 年 | 完整Quartz格式 |
| 8位 | 秒 分 时 日 月 周 ? 年 | Spring扩展格式 |

### 6.2 解析器实现

```java
public class CronParser {
    // 解析表达式，返回下一次触发时间
    public List<LocalDateTime> getNextExecutions(String cron, int count);

    // 校验表达式是否合法
    public boolean isValid(String cron);

    // 转换为标准6位格式
    public String normalize(String cron);
}
```

## 7. 前端 API 对接

### 7.1 API 接口定义 (frontend/src/api/index.ts)

```typescript
// 创建任务
export function createTask(data: CreateTaskRequest) {
  return request.post('/tasks', data);
}

// 更新任务
export function updateTask(id: number, data: UpdateTaskRequest) {
  return request.put(`/tasks/${id}`, data);
}

// 获取任务详情
export function getTaskDetail(id: number) {
  return request.get(`/tasks/${id}`);
}

// 删除任务
export function deleteTask(id: number) {
  return request.delete(`/tasks/${id}`);
}

// 执行任务
export function executeTask(id: number) {
  return request.post(`/tasks/${id}/execute`);
}

// 校验Cron表达式
export function validateCron(cron: string) {
  return request.post('/tasks/validate-cron', { cron });
}
```

## 9. 约束与边界

1. **脚本存储**：本地磁盘，路径可配置
2. **Cron表达式**：支持5-8位，自动规范化存储
3. **触发配置**：JSON格式存储，解析后注册调度
4. **版本管理**：每次更新脚本自动创建新版本
5. **执行方式**：Python脚本由独立执行器运行（Docker/进程）
