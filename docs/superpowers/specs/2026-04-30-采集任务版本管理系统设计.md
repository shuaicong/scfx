# 采集任务版本管理系统设计

## 1. 需求概述

对采集脚本进行版本管理，支持：
- 每次保存自动创建版本
- 时间线视图展示版本历史
- 任意版本对比
- 回滚到任意历史版本
- 可补充版本变更说明

## 2. 数据库设计

### 2.1 版本表 t_script_version

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键 |
| script_id | BIGINT | 关联脚本ID |
| version | INT | 版本号(自增) |
| content | TEXT | 版本内容快照 |
| change_description | VARCHAR(500) | 变更说明(可选) |
| created_by | VARCHAR(50) | 操作人 |
| created_at | TIMESTAMP | 创建时间 |

### 2.2 关联关系

- 一个脚本 → 多个版本
- 版本号在脚本ID内自增

## 3. 后端API设计

### 3.1 接口列表

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /scripts/{id}/versions | 获取版本列表(时间线) |
| GET | /scripts/{id}/versions/{vid} | 获取特定版本内容 |
| GET | /scripts/{id}/diff | 对比两个版本 ?v1=x&v2=y |
| POST | /scripts/{id}/rollback/{vid} | 回滚到指定版本 |
| PUT | /scripts/{id}/versions/{vid} | 更新版本说明 |

### 3.2 核心逻辑

**保存时创建版本：**
```
1. 保存脚本内容到文件
2. 创建新版本记录(content=新内容, version=max+1)
3. 返回新版本号
```

**回滚流程：**
```
1. 读取目标版本内容
2. 更新脚本文件为目标版本内容
3. 创建新版本(内容=目标版本, description="回滚到v{x}")
4. 返回新版本号
```

## 4. 前端设计

### 4.1 时间线视图

```
● v5 (最新) - 2026-04-30 10:30 - 修复登录问题
│
● v4 - 2026-04-29 15:20 - 优化爬取逻辑
│
● v3 - 2026-04-28 09:00 - 首次上线
```

### 4.2 版本卡片(点击展开)

- 完整代码内容
- 与上一版本的差异高亮(diff)
- 操作按钮：对比 | 回滚 | 删除

### 4.3 组件结构

```
Scripts.vue
├── VersionTimeline.vue      # 左侧版本时间线
├── VersionDetail.vue        # 右侧版本内容
├── VersionDiff.vue          # diff对比弹窗
└── VersionHistoryDrawer.vue # 版本历史抽屉
```

## 5. 版本对比设计

- 使用左右并排diff视图
- 绿色高亮新增行，红色高亮删除行
- 顶部显示两个版本号和时间

## 6. 实现步骤

1. 创建数据库版本表
2. 实现ScriptVersionService
3. 扩展CollectionScriptController接口
4. 创建前端VersionTimeline组件
5. 创建VersionDiff对比组件
6. 集成到Scripts.vue

## 7. 约束与边界

- 版本内容存储完整快照，非diff delta
- 回滚操作本身产生新版本
- 不支持删除单个版本(防止版本链断裂)
