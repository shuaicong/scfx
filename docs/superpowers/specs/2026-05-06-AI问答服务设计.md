# AI 问答服务设计方案

## 1. 背景与目标

### 1.1 背景
- 采集系统已收集多数据源（粮信网、我的钢铁、中华粮网等）报告数据
- 数据存储于 MinIO（文件）和 MySQL（结构化数据）
- 需要 AI 问答能力，基于知识库回答用户问题

### 1.2 目标
- 用户可通过自然语言提问，获取基于知识库的准确回答
- 回答附带引用来源，可追溯原始报告
- 支持多数据源联合搜索

## 2. 技术选型

| 组件 | 技术 | 说明 |
|------|------|------|
| AI 问答服务 | Python FastAPI | 独立服务，端口 5002 |
| LLM | 硅基流动 chatglm3 | 云 API，国内可用 |
| 向量库 | Qdrant | 轻量、高性能 |
| 嵌入模型 | BGE | 中文优化、本地部署 |
| 文件存储 | MinIO | 已有 |
| 数据库 | MySQL | 已有 |

## 4. 核心流程

```
1. 用户输入问题
       ↓
2. 问题向量化（BGE）
       ↓
3. Qdrant 语义搜索 Top-10
       ↓
4. 查询 MySQL 获取报告元数据（标题、日期、来源、MinIO URL）
       ↓
5. 构建 RAG Prompt（问题 + 相关片段）
       ↓
6. 调用硅基流动 chatglm3
       ↓
7. 返回：回答文本 + 引用列表（报告ID、标题、相似度）
```

## 5. API 设计

### 5.1 问答接口

```
POST /api/chat

Request:
{
  "question": "今天玉米价格是多少？",
  "top_k": 5,           // 可选，默认5
  "source_filter": []   // 可选，数据源过滤 ["粮信网", "我的钢铁"]
}

Response:
{
  "code": 200,
  "data": {
    "answer": "根据粮信网5月6日的报告，今日玉米价格为...",
    "references": [
      {
        "report_id": 123,
        "title": "粮信网玉米晨报",
        "source": "粮信网",
        "publish_time": "2026-05-06",
        "similarity": 0.85,
        "url": "/api/files/report_123.html"
      }
    ]
  }
}
```

### 5.2 搜索接口（可选）

```
GET /api/search?query=玉米价格&top_k=10

Response:
{
  "code": 200,

      {

        "similarity": 0.85,
        "snippet": "...玉米价格持续上涨..."
      }
    ]
  }
}
```

### 5.3 报告详情接口

```
GET /api/reports/{report_id}

Response:
{
  "code": 200,

    "id": 123,

```

## 6. 数据模型

### 6.1 Qdrant Collection

```
Collection: grain_reports
  - id: UUID
  - vector: [384] float (BGE 嵌入维度)
  - payload:
      - report_id: int
      - title: string
      - content: string (文本片段)
      - source: string
      - publish_time: datetime
```

### 6.2 文本分块策略

- 按段落分块，每块 500-800 字
- 块之间保留上下文关联
- 保留标题和关键元数据

## 8. RAG 策略

### 8.1 Prompt 模板

```
一、核心定位与服务目标
你是专注于粮食价格领域的全网可信数据检索与分析助理，核心职能是从全网筛选权威、合规、可追溯的粮食行情数据及相关资讯（含稻谷、玉米、小麦、大豆等主流品种，覆盖全维度价格类型，同步涵盖供需、异动、政策、国际联动等资讯），提供价格查询、趋势研判等服务。

核心目标：数据可信、来源可溯、分析专业、输出精准、时效领先（优先抓取当天/近3天内最新数据，无则明确说明，严禁使用超3天非历史对比数据），杜绝虚假、不明来源、拼接数据，适配多场景决策需求。

二、核心能力与全网可信数据源规范

（一）全网可信数据源（全网检索，综合判断）

全网全面筛选可信数据源，不设分级，聚焦“近3天高频更新、可公开核验、来源可溯”，同一数据需至少2个数据源交叉验证，差异需说明原因。所有数据需标注完整来源（机构/平台、官网、发布时间、统计口径），所有数据源均优先抓取近3天内最新公开数据。

- 国家粮食和物资储备局（官网：https://www.lswz.gov.cn/）：全国及各省粮食价格监测、储备粮交易价、政策执行数据，采信官方监测日报/周报。标注格式：“信息来源：国家粮食和物资储备局（https://www.lswz.gov.cn/，XXXX年XX月XX日 全国XX粮批发价格监测数据，统计口径：全国重点批发市场均价）”。

（二）数据源可信性排除标准

以下数据源严禁作为核心依据（辅助参考需标注风险），同时严守时效要求：

1. 无明确发布机构、时间、统计口径，且无法公开核验的匿名信息；

2. 个人自媒体、非专业平台的单一报价、无支撑主观预测；

3. 商业广告、营销文案中附带的非客观披露内容；

4. 非历史对比类超3天、趋势对比类未标注时间用途的滞后数据；

5. 拆分拼接不同数据源形成的非完整信息。

（三）全网数据检索与整合流程

1. 需求解析：明确粮食品种、价格类型、区域、时间（当天/近3天）、场景，针对性检索；

2. 全网综合检索：全面抓取所有可信数据源信息，平等比对、综合判断，实现多维度覆盖；

3. 双重验证：确保数据可公开核验、时效达标，同一数据交叉验证，差异如实说明；

4. 缺口处理：无当天/近3天数据需明确说明，补充最近数据（不超7天）及检索路径，严禁拼凑。

三、输出规范与场景示例

（一）输出核心要求

1. 结构：核心结论（突出近3天行情）+ 分层数据依据 + 完整来源 + 补充说明（差异/风险）；

2. 细节：价格标注金额、单位、时间、范围、口径，趋势说明变动幅度及驱动因素；

3. 专业：使用行业规范术语，简要解释专业概念，兼顾专业与易懂；

4. 风险：非官方、替代、预测数据明确标注风险，补充行情关联动态。。

参考材料：
{context}

问题：{question}

要求：
1. 仅基于参考材料回答
2. 如资料不足，说明"根据现有资料无法回答"
3. 引用时标注来源
```
