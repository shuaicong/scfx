# 采集任务可视化管理系统设计

## 1. 概述

### 1.1 项目背景
海南储备集团粮食市场智能分析平台需要采集多个网站的数据（粮信网、我的钢铁网、中华粮网等），现有系统缺乏统一的采集任务可视化管理界面。

### 1.2 设计目标
1. 采集任务可视化：清楚知道每个采集任务的执行结果、状态、日志
2. 灵活调度配置：支持单次触发、周期触发（每天/每周/每月）、Cron 表达式
3. 版本管理：脚本和配置的完整版本历史

## 2. 功能模块

### 2.1 页面结构
- `/collection/tasks` - 采集任务列表页
- `/collection/tasks/:id` - 任务详情页
- `/collection/tasks/:id/versions` - 版本历史页
- `/collection/tasks/:id/executions/:execId` - 执行详情页

### 2.3 任务详情页
- 基本信息：数据源、触发方式、状态
- 执行统计：执行次数、成功/失败次数、成功率、平均时长
- 执行历史：每次执行的记录列表
- 脚本版本：版本历史时间线
- 采集数据：查看采集的数据（可选）

#### 立即执行功能
- 点击"立即执行"按钮
- 显示执行进度：执行时长、实时日志
- 支持取消执行
- 执行完成后显示结果

#### 执行状态流转
```
PENDING → RUNNING → SUCCESS/FAILED/CANCELLED → DONE
```

## 3. 采集时间配置

### 3.1 触发类型
- 单次触发 - 指定日期时间执行一次
- 周期触发 - 每天/每周/每月循环执行
- Cron 表达式 - 自定义 cron 表达式

### 3.2 单次触发
```
执行时间: [📅 2026-04-29]  [08: ] : [00 ] : [00 ]

结束时间: ( ) 永不结束
         (●) 于指定日结束  [📅 2026-05-29]
         ( ) 重复 N 次后结束  [____] 次
```

### 3.3 周期触发 - 每日
```

  [08 ] 时 [00 ] 分 [00 ] 秒
```

### 3.4 周期触发 - 每周
```

  [08 ] 时 [00 ] 分 [00 ] 秒

  ☑ 周一   ☑ 周二   ☑ 周三   ☑ 周四   ☑ 周五   ☐ 周六   ☐ 周日

### 3.5 周期触发 - 每月
```
重复: ( ) 每天   ( ) 每周   (●) 每月

触发时间:
  [08 ] 时 [00 ] 分 [00 ] 秒

## 6. 数据模型

### 6.1 采集脚本表 (collection_script)
```sql
CREATE TABLE collection_script (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    script_name VARCHAR(200) NOT NULL,
    description VARCHAR(500),
    script_path VARCHAR(500),
    script_content TEXT,

    -- 触发配置
    trigger_type VARCHAR(20),                  -- once/repeat/cron
    repeat_type VARCHAR(20),                   -- daily/weekly/monthly
    repeat_time VARCHAR(10),                   -- HH:mm:ss
    weekly_days VARCHAR(20),                   -- 1,3,5 (多选)
    monthly_day INT,
    monthly_last_day BOOLEAN,                  -- 是否每月最后一天
    cron_expression VARCHAR(100),

    -- 结束条件
    end_type VARCHAR(20),                     -- never/date/count
    end_time DATETIME,
    repeat_count INT,

    -- 状态
    is_enabled BOOLEAN DEFAULT TRUE,
    current_version INT DEFAULT 1,

    -- 执行统计
    execution_count INT DEFAULT 0,
    success_count INT DEFAULT 0,
    failed_count INT DEFAULT 0,
    last_execution_time DATETIME,
    next_execution_time DATETIME,
    avg_duration_ms BIGINT,

    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

### 6.2 脚本版本表 (script_version)
```sql
CREATE TABLE script_version (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    script_id BIGINT NOT NULL,
    version_num INT NOT NULL,

    -- 脚本快照
    script_content TEXT NOT NULL,
    script_name VARCHAR(200),

    -- 配置快照
    trigger_type VARCHAR(20),
    repeat_type VARCHAR(20),
    repeat_time VARCHAR(10),
    weekly_days VARCHAR(20),
    monthly_day INT,
    monthly_last_day BOOLEAN,
    cron_expression VARCHAR(100),
    end_type VARCHAR(20),
    end_time DATETIME,
    repeat_count INT,

    -- 元信息
    change_description VARCHAR(500) NOT NULL,
    created_by VARCHAR(50),
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    is_current BOOLEAN DEFAULT FALSE,

    -- 执行统计
    execution_count INT DEFAULT 0,
    success_count INT DEFAULT 0,
    failed_count INT DEFAULT 0,

    INDEX idx_script_id (script_id),
    INDEX idx_created_at (created_at)
);
```

### 6.3 任务执行表 (task_execution)
```sql
CREATE TABLE task_execution (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    execution_id VARCHAR(64) UNIQUE NOT NULL,
    script_id BIGINT NOT NULL,
    version_id BIGINT,

    trigger_type VARCHAR(20) NOT NULL,         -- manual/scheduled/api
    status VARCHAR(20) NOT NULL,              -- pending/running/success/failed/cancelled

    start_time DATETIME(3),
    end_time DATETIME(3),
    duration_ms BIGINT,
    error_message TEXT,

    INDEX idx_script_id (script_id),
    INDEX idx_status (status),
    INDEX idx_start_time (start_time)
);
```

### 6.4 执行日志表 (task_execution_log)
```sql
CREATE TABLE task_execution_log (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    execution_id VARCHAR(64) NOT NULL,
    script_id BIGINT,
    level VARCHAR(10) DEFAULT 'INFO',          -- DEBUG, INFO, WARN, ERROR
    message TEXT,
    timestamp DATETIME(3) NOT NULL,
    INDEX idx_execution_id (execution_id),
    INDEX idx_timestamp (timestamp)
);
```

## 7. API 接口

### 7.1 脚本管理
```
GET    /api/collection/scripts              # 列表
POST   /api/collection/scripts              # 创建
GET    /api/collection/scripts/{id}         # 详情
PUT    /api/collection/scripts/{id}         # 更新
DELETE /api/collection/scripts/{id}         # 删除
POST   /api/collection/scripts/{id}/enable  # 启用
POST   /api/collection/scripts/{id}/disable # 禁用
POST   /api/collection/scripts/{id}/execute # 立即执行
```

### 7.2 版本管理
```
GET    /api/collection/scripts/{scriptId}/versions      # 版本列表
GET    /api/collection/scripts/{scriptId}/versions/{id} # 版本详情
POST   /api/collection/scripts/{scriptId}/versions/{id}/restore # 恢复版本
GET    /api/collection/scripts/{scriptId}/versions/compare # 版本对比
```

### 7.3 执行记录
```
GET    /api/collection/scripts/{scriptId}/executions   # 执行列表
GET    /api/collection/executions/{executionId}       # 执行详情
GET    /api/collection/executions/{executionId}/logs   # 执行日志
POST   /api/collection/executions/{executionId}/cancel  # 取消执行
```

### 7.4 实时日志
```
WebSocket /ws/execution/{executionId}/logs  # 实时日志推送
```

## 8. 实现优先级

### Phase 1 - 核心功能
1. 任务列表页（启停、筛选）
2. 任务详情页（基本信息、配置展示）
3. 执行记录（列表、详情、日志）
4. 立即执行功能

### Phase 2 - 调度功能
1. 时间配置（单次/周期/Cron）
2. 定时调度执行
3. 未来触发时间预览

### Phase 3 - 版本管理
1. 版本历史时间线
2. 版本详情查看
3. 版本对比
4. 保存时记录修改说明

### Phase 4 - 增强功能
1. 实时日志 WebSocket
2. 批量操作
3. 执行统计图表
