# 采集管理系统对比功能设计

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 为执行详情页提供三种对比功能，支持执行记录、脚本版本、执行日志的对比分析

---

## 一、功能概述

| 功能 | 入口 | 用途 |
|------|------|------|
| 对比执行记录 | 执行详情页 → [对比执行记录] | 对比不同执行的采集数量、耗时、状态 |
| 对比脚本版本 | 执行详情页 → [对比脚本版本] | 对比两个版本脚本的内容差异 |
| 对比执行日志 | 执行详情页 → [对比执行日志] | 对比本次与上次执行的日志差异 |

---

## 二、对比执行记录

### 2.1 功能说明

从当前任务的历史执行记录中选择两条进行对比，快速发现执行结果的变化。

### 2.2 交互流程

```
执行详情页点击 [对比执行记录]
    ↓
弹出对比选择弹窗
    ↓
选择两条执行记录
    ↓
确认对比
    ↓
显示并排对比视图
```

### 2.3 选择弹窗

```
┌─────────────────────────────────────────────────────────────────────┐
│ 选择执行记录                                              [X]        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  选择要对比的两条执行记录：                                          │
│                                                                     │
│  ○  2026-05-15 09:30  ✓ 成功  采集3条  耗时45秒  v3               │
│  ○  2026-05-14 09:30  ✓ 成功  采集5条  耗时38秒  v2               │
│  ○  2026-05-13 09:30  ⚠ 失败  采集0条  耗时30秒  v2               │
│  ○  2026-05-12 09:30  ✓ 成功  采集4条  耗时52秒  v1               │
│                                                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                            [取消]  [确认对比]        │
└─────────────────────────────────────────────────────────────────────┘
```

### 2.4 对比视图

```

│ 执行记录对比                                 [导出对比] [关闭]           │
├─────────────────────────────────────────────────────────────────────────┤

│  ┌─────────────────────┬─────────────────────┐                          │
```

### 2.5 接口设计

**获取执行记录列表：**
```
GET /scripts/{scriptId}/executions
Response: [
  { executionId, startTime, status, collectedCount, duration, scriptVersion }
]
```

**对比执行记录：**
```
GET /scripts/{scriptId}/executions/compare?executionId1=xxx&executionId2=xxx
Response: {
  "record1": { startTime, status, collectedCount, duration, scriptVersion },
  "record2": { startTime, status, collectedCount, duration, scriptVersion },
  "changes": {
    "collectedCount": { diff: -2, percent: -40 },
    "duration": { diff: 7, percent: 18 }
  }
}
```

## 三、对比脚本版本

### 3.1 功能说明

查看当前脚本的两个版本之间的内容差异，用于分析脚本修改导致的问题。

### 3.2 交互流程

```
执行详情页点击 [对比脚本版本]
    ↓
弹出版本选择弹窗
    ↓
选择两个版本（默认本次和上次）
    ↓
确认对比
    ↓
显示版本差异视图
```

### 3.3 选择弹窗

```
┌─────────────────────────────────────────────────────────────────────┐
│ 选择脚本版本                                              [X]        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  选择要对比的两个版本：                                              │
│                                                                     │
│  ● v3 (当前)  2026-05-15 14:30  上传 by admin                       │
│  ○ v2        2026-05-14 10:00  上传 by admin                       │
│  ○ v1        2026-05-10 09:00  上传 by admin                       │
│                                                                     │
│  [ ] 显示行号差异                                                     │
│  [ ] 仅显示有变化的行                                                 │
│                                                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                            [取消]  [确认对比]        │
└─────────────────────────────────────────────────────────────────────┘
```

### 3.4 对比视图

```
┌─────────────────────────────────────────────────────────────────────────┐
│ 脚本版本对比                              [v3 vs v2] [导出] [关闭]       │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  ┌────────────────────┬────────────────────┐                            │
│  │ v3 (2026-05-15)    │ v2 (2026-05-14)    │                            │
│  ├────────────────────┼────────────────────┤                            │
│  │  1 │ # -*- coding  │  1 │ # -*- coding  │                            │
│  │  2 │ import reques │  2 │ import reques │                            │
│  │  3 │               │  3 │ - # TODO: 删  │  ← 删除                   │
│  │  4 │ + # 新增日志  │  4 │ + # 新增日志  │  ← 新增                   │
│  │  5 │ class Coll    │  5 │ class Coll    │                            │
│  │  ...               │  ...               │                            │
│  └────────────────────┴────────────────────┘                            │
│                                                                         │
│  差异统计：新增 3 行，删除 1 行，修改 2 行                               │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

**差异高亮规则：**
- 绿色背景：新增的行
- 红色背景：删除的行
- 黄色背景：修改的行

### 3.5 接口设计

**获取脚本版本列表：**
```
GET /datasource/{code}/versions
Response: [
  { version: 3, createdAt, createdBy, md5 },
  { version: 2, createdAt, createdBy, md5 },
  { version: 1, createdAt, createdBy, md5 }
]
```

**获取版本内容：**
```
GET /datasource/{code}/versions/{version}/content
Response: { content: "..." }
```

**对比两个版本：**
```
GET /datasource/{code}/versions/compare?v1=3&v2=2
Response: {
  "v1": { version: 3, content: "...", createdAt: "..." },
  "v2": { version: 2, content: "...", createdAt: "..." },
  "diff": [
    { type: "delete", line: 3, content: "- # TODO: 删" },
    { type: "add", line: 4, content: "+ # 新增日志" }
  ],
  "stats": { added: 3, deleted: 1, modified: 2 }
}
```

## 四、对比执行日志

### 4.1 功能说明

对比本次执行与上次执行的日志内容，快速定位执行过程中哪里出现了问题或变化。

### 4.2 交互流程

```
执行详情页点击 [对比执行日志]
    ↓
直接加载本次和上次执行的日志
    ↓
显示并排对比视图（同一时间线对齐）
```

### 4.3 对比视图

```
┌─────────────────────────────────────────────────────────────────────────┐
│ 执行日志对比                    本次: 2026-05-15  vs  上次: 2026-05-14  │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  ┌──────────────────────────┬──────────────────────────┐               │
│  │ 2026-05-15 09:30         │ 2026-05-14 09:30         │               │
│  ├──────────────────────────┼──────────────────────────┤               │
│  │ [08:00:01] INFO  开始执行 │ [08:00:01] INFO  开始执行 │ ✓ 相同        │
│  │ [08:00:02] INFO  登录成功 │ [08:00:02] INFO  登录成功 │ ✓ 相同        │
│  │ [08:00:03] INFO  发现页面 │ [08:00:03] INFO  发现页面 │ ✓ 相同        │
│  │ [08:00:04] ERROR 元素未找到│ [08:00:04] WARN 元素偏移  │ ⚠ 差异        │
│  │ [08:00:05] ERROR 重试失败  │ [08:00:05] INFO  重试成功  │ ✗ 差异        │
│  │ [08:00:06] INFO  跳过      │ [08:00:06] INFO  采集完成  │ ✗ 差异        │
│  └──────────────────────────┴──────────────────────────┘               │
│                                                                         │
│  差异标记说明：                                                         │
│  ✓ 相同 - 两边日志一致                                                 │
│  ⚠ 差异 - 日志内容不同但都成功                                          │
│  ✗ 差异 - 一边成功一边失败，需要关注                                     │
│                                                                         │
│  时间轴对齐：按时间顺序排列，时间戳相同的行对齐显示                       │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 4.4 差异分析

```
差异统计：
├─ 相同日志：8 条
├─ 警告差异：1 条 (元素偏移)
└─ 失败差异：2 条 (重试失败 vs 重试成功)

首次出现差异：08:00:04 [ERROR 元素未找到 vs WARN 元素偏移]
可能原因：
1. 网站结构变化
2. 网络波动导致超时
3. 页面元素定位失效

建议检查：
1. 查看目标网站是否改版
2. 检查网络连接稳定性
3. 更新元素选择器
```

### 4.5 接口设计

**获取执行日志：**
```
GET /executions/{executionId}/logs
Response: [
  { timestamp: "08:00:01", level: "INFO", message: "开始执行" },
  { timestamp: "08:00:02", level: "INFO", message: "登录成功" }
]
```

**对比两次执行的日志：**
```
GET /executions/{executionId}/logs/compare-with-last
Response: {
  "current": { executionId, startTime, logs: [...] },
  "last": { executionId, startTime, logs: [...] },
  "aligned": [
    {
      "time1": "08:00:01", "time2": "08:00:01",
      "level1": "INFO", "level2": "INFO",
      "msg1": "开始执行", "msg2": "开始执行",
      "diff": false
    },
    {
      "time1": "08:00:04", "time2": "08:00:04",
      "level1": "ERROR", "level2": "WARN",
      "msg1": "元素未找到", "msg2": "元素偏移",
      "diff": true, "severity": "error"
    }
  ],
  "stats": { same: 8, warning: 1, error: 2 }
}
```

## 七、验证方案

1. **对比执行记录**
   - 选择两条执行记录，对比视图正确显示差异
   - 变化分析数字正确

2. **对比脚本版本**
   - 选择两个版本，差异视图正确显示
   - 新增/删除行高亮正确

3. **对比执行日志**
   - 同一时间线的日志对齐显示
   - 差异标记正确（✓/⚠/✗）
