# 采集任务版本管理系统 - 系统设计

## 1. 系统概述

### 1.1 背景
采集脚本需要频繁修改和版本管理，每次保存应自动创建版本，支持版本历史查看、对比和回滚。

### 1.2 功能范围
- 每次保存自动创建版本（快照完整存储）
- 时间线视图展示版本历史
- 任意两版本对比（高亮差异）
- 回滚到任意历史版本（回滚操作本身产生新版本）
- 可补充版本变更说明

## 3. 数据库设计

### 3.1 版本表 t_script_version

```sql
CREATE TABLE IF NOT EXISTS t_script_version (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    script_id BIGINT NOT NULL,              -- 关联脚本ID
    version_num INT NOT NULL,               -- 版本号(自增,基于script_id)
    script_name VARCHAR(100),               -- 脚本名称快照
    script_content TEXT,                    -- 脚本内容完整快照
    trigger_type VARCHAR(20),               -- 触发类型快照
    repeat_type VARCHAR(20),                -- 周期类型快照
    repeat_time VARCHAR(20),                -- 周期时间快照
    weekly_days VARCHAR(100),               -- 每周天数快照
    monthly_day INT,                        -- 每月日期快照
    monthly_last_day BOOLEAN,               -- 每月最后一天快照
    cron_expression VARCHAR(100),           -- Cron表达式快照
    end_type VARCHAR(20),                   -- 结束条件快照
    end_time TIMESTAMP,                     -- 结束时间快照
    repeat_count INT,                       -- 重复次数快照
    change_description VARCHAR(500),        -- 变更说明(可选)
    created_by VARCHAR(50),                 -- 操作人
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    is_current BOOLEAN DEFAULT FALSE        -- 是否当前版本
);
```

### 3.2 关联关系
- 一个 `CollectionScript` → 多个 `ScriptVersion`
- 版本号在脚本ID内自增（跨脚本独立编号）
- `is_current` 标记当前生效版本

## 4. API 设计

### 4.1 接口列表

| 方法 | 路径 | 说明 | 请求体/参数 |
|------|------|------|------------|
| GET | `/scripts/{id}/versions` | 获取版本列表(时间线) | page, size |
| GET | `/scripts/{id}/versions/{vid}` | 获取特定版本内容 | - |
| GET | `/scripts/{id}/diff?v1=x&v2=y` | 对比两个版本 | v1, v2 版本ID |
| POST | `/scripts/{id}/versions` | 创建新版本 | changeDescription |
| PUT | `/scripts/{id}/versions/{vid}` | 更新版本说明 | changeDescription |
| POST | `/scripts/{id}/rollback/{vid}` | 回滚到指定版本 | - |
| DELETE | `/scripts/{id}/versions/{vid}` | 删除版本(仅非当前版本) | - |

### 4.2 接口详情

#### 获取版本列表
```
GET /scripts/{scriptId}/versions

Response:
{
  "code": 200,
  "data": {
    "records": [
      {
        "id": 5,
        "scriptId": 1,
        "versionNum": 5,
        "changeDescription": "修复登录问题",
        "createdBy": "admin",
        "createdAt": "2026-04-30T10:30:00",
        "isCurrent": true
      },
      {
        "id": 4,
        "scriptId": 1,
        "versionNum": 4,
        "changeDescription": "优化爬取逻辑",
        "createdBy": "admin",
        "createdAt": "2026-04-29T15:20:00",
        "isCurrent": false
      }
    ],
    "total": 5,
    "page": 1,
    "size": 20
  }
}
```

#### 版本对比
```
GET /scripts/{scriptId}/diff?v1=4&v2=5

Response:
{
  "code": 200,
  "data": {
    "version1": { "id": 4, "versionNum": 4, "scriptContent": "...", ... },
    "version2": { "id": 5, "versionNum": 5, "scriptContent": "...", ... },
    "diff": {
      "scriptContent": ["old content", "new content"],
      "triggerType": ["once", "cycle"]
    }
  }
}
```

#### 回滚版本
```
POST /scripts/{scriptId}/rollback/{versionId}

Response:
{
  "code": 200,
  "data": {
    "id": 6,
    "versionNum": 6,
    "changeDescription": "回滚到v3",
    "isCurrent": true
  }
}
```
回滚操作流程：
1. 读取目标版本内容
2. 更新脚本文件
3. 创建新版本（内容=目标版本，说明="回滚到v{x}"）
4. 返回新版本号

## 6. 核心流程

### 6.1 保存时创建版本
```
1. 保存脚本内容到文件
2. 创建新版本记录（content=新内容, version=max+1）
3. 返回新版本号
```

### 6.2 回滚流程
```
1. 读取目标版本内容
2. 更新脚本文件为目标版本内容
3. 创建新版本（内容=目标版本, description="回滚到v{x}"）
4. 返回新版本号
```

---

## 7. 约束与边界

1. **版本内容存储完整快照**，非 diff delta（便于直接查看任意版本）
2. **回滚操作本身产生新版本**（不修改历史，保持版本链完整）
3. **删除版本仅允许非当前版本**（防止版本链断裂）
4. **版本号在脚本ID内自增**（不同脚本版本号独立）
