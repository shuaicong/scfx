# 知识库系统设计

## 1. 系统概述

### 1.1 背景
采集系统已收集多数据源（粮信网、我的钢铁、中华粮网等）报告数据，需构建统一知识库管理系统，为 AI 问答提供知识支撑。

### 1.2 目标
- 统一管理采集数据、用户上传文档、人工录入文本
- 提供文本向量化存储和语义检索能力
- 为 AI 问答提供知识查询服务

### 1.3 功能范围
| 功能 | 说明 |
|------|------|
| 采集数据接入 | 接收 Java 后端推送的采集报告，自动解析、分块、向量化 |
| 文档上传 | 用户上传 PDF/Word/TXT 文件，系统自动解析文本并入库 |
| 人工录入 | 用户手动输入文本片段，标注来源后存入知识库 |
| 知识检索 | 语义检索，支持按来源/类型筛选，返回相关片段及相似度 |
| 知识管理 | 知识的查看、编辑、删除、重新向量化 |
| 知识统计 | 知识库总量、来源分布、向量状态统计 |

## 2. 技术架构

### 2.1 技术栈

| 组件 | 技术 | 说明 |
|------|------|------|
| 知识库服务 | Python FastAPI | 独立服务，端口 5002 |
| 向量数据库 | Qdrant | 轻量、高性能向量检索 |
| 嵌入模型 | BGE | 中文优化、本地部署 |
| 元数据库 | MySQL | 已有，存储知识元数据 |
| 文件存储 | MinIO | 已有，存储上传文档 |
| 文档解析 | pdfplumber / python-docx | Python PDF/Word 解析 |

## 3. 数据模型

### 3.1 MySQL 表: t_knowledge_base

```sql
CREATE TABLE IF NOT EXISTS t_knowledge_base (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    title VARCHAR(500) NOT NULL,           -- 知识标题
    source_type VARCHAR(20) NOT NULL,      -- collection/upload/manual
    source_name VARCHAR(100),              -- 来源: 粮信网/我的钢铁/用户上传/人工录入
    original_url VARCHAR(1000),            -- 原文链接
    author VARCHAR(100),                  -- 作者/来源
    publish_time DATETIME,                 -- 发布时间
    content TEXT NOT NULL,                 -- 原始文本内容
    content_hash VARCHAR(64),              -- 内容 SHA256 (去重用)
    file_path VARCHAR(500),                -- MinIO 文件路径
    file_type VARCHAR(20),                 -- pdf/word/txt/html
    chunk_count INT DEFAULT 0,            -- 分块数量
    vector_status VARCHAR(20) DEFAULT 'pending',  -- pending/processing/completed/failed
    vector_ids VARCHAR(500),               -- Qdrant vector IDs (逗号分隔)
    execution_id VARCHAR(50),              -- 关联采集执行ID
    created_by VARCHAR(50),                -- 创建人
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    deleted INT DEFAULT 0,
    INDEX idx_source_type (source_type),
    INDEX idx_source_name (source_name),
    INDEX idx_vector_status (vector_status),
    INDEX idx_content_hash (content_hash)
);
```

### 4.5 知识管理

```
GET /api/knowledge
  - 知识列表 (分页)
  - Query: page, size, sourceType, sourceName, vectorStatus

GET /api/knowledge/{id}
  - 知识详情 (包含分块内容)

PUT /api/knowledge/{id}
  - 更新知识 (title, content)
  - 注意: 更新内容需要重新向量化

DELETE /api/knowledge/{id}
  - 删除知识 (软删除, 同时删除 Qdrant 向量)

POST /api/knowledge/{id}/revectorize
  - 重新向量化指定知识

GET /api/knowledge/stats
  - 知识库统计
  - Response: { total, bySource, byStatus, totalChunks }
```

## 5. 处理流程

### 5.1 文档上传处理流程

```
用户上传 PDF
      │
      ▼
┌─────────────────────────────────────────┐
│  1. 文件存储                            │
│     保存到 MinIO: /knowledge/{kbId}/原始文件 │
└─────────────────────────────────────────┘
      │
      ▼
┌─────────────────────────────────────────┐
│  2. 文档解析 (Python)                   │
│     ├─ PDF: pdfplumber 提取文本         │
│     ├─ Word: python-docx 提取文本        │
│     └─ TXT: 直接读取                    │
└─────────────────────────────────────────┘
      │
      ▼
┌─────────────────────────────────────────┐
│  3. 文本清洗                            │
│     ├─ 去除特殊字符                     │
│     ├─ 标准化空白字符                   │
│     └─ 生成 content_hash (去重)         │
└─────────────────────────────────────────┘
      │
      ▼
┌─────────────────────────────────────────┐
│  4. 重复检测                            │
│     ├─ 计算 content_hash                │
│     ├─ 查询 MySQL 是否存在              │
│     └─ 存在则跳过，标记为重复           │
└─────────────────────────────────────────┘
      │
      ▼
┌─────────────────────────────────────────┐
│  5. 文本分块                            │
│     ├─ 按段落分块                       │
│     ├─ 每块 500-800 字                 │
│     ├─ 块之间保留上下文                 │
│     └─ 记录 chunk_count                 │
└─────────────────────────────────────────┘
      │
      ▼
┌─────────────────────────────────────────┐
│  6. 向量化 (BGE)                        │
│     ├─ 每块调用 BGE 嵌入                │
│     ├─ 存储到 Qdrant                    │
│     └─ 保存 vector_ids                  │
└─────────────────────────────────────────┘
      │
      ▼
┌─────────────────────────────────────────┐
│  7. 更新状态                            │
│     ├─ 更新 MySQL vector_status         │
│     └─ 通知前端处理完成                 │
└─────────────────────────────────────────┘
```

## 9. 与现有系统对接

### 9.1 采集系统 → 知识库

```
采集完成 (Python)
    │
    ├─ 保存截图/HTML → MinIO
    ├─ 保存报告 → MySQL (已有 t_report)
    └─ 调用 /api/knowledge/ingest
            │
            └─→ 知识库自动向量化
```

### 9.2 Java 后端 → 知识库

```java
// CollectionService.java - 采集完成后调用知识库接入
@Autowired
private RestTemplate restTemplate;

public void afterCollectionComplete(String executionId, List<Report> reports) {
    // 1. 保存报告到 MySQL (已有逻辑)
    reportRepository.saveAll(reports);

    // 2. 调用知识库接入接口
    String url = "http://ai-qa-service:5002/api/knowledge/ingest";
    Map<String, Object> payload = new HashMap<>();
    payload.put("executionId", executionId);
    payload.put("source", "liangxinwang");
    payload.put("reports", reports.stream().map(r -> {
        Map<String, Object> item = new HashMap<>();
        item.put("title", r.getTitle());
        item.put("source", r.getSource());
        item.put("url", r.getOriginalUrl());
        item.put("content", r.getContent());
        item.put("publishTime", r.getPublishTime());
        return item;
    }).collect(Collectors.toList()));

    restTemplate.postForObject(url, payload, Map.class);
}
```
